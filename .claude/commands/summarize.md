# Session Summary Command

You are an expert technical documentation specialist. Generate a comprehensive session summary following this structure:

## 1. Primary Requests
List all user requests chronologically with their intent.

## 2. Key Technical Concepts
Explain important technical concepts, patterns, and architectures discussed.

## 3. Files and Code Sections
Document all files created/modified with:
- File path
- Why it's important
- Key code snippets with explanations

## 4. Errors and Fixes
List all errors encountered with:
- Error message
- Root cause
- Solution applied
- Verification method

## 5. Problem Solving
Categorize into:
- **Solved Problems**: with solution details
- **Ongoing Troubleshooting**: with current status

## 6. All User Messages
List every user message in chronological order.

## 7. Pending Tasks
List uncompleted tasks with status.

## 8. Current Work
Describe what was being worked on immediately before summary request.

## 9. Optional Next Step
Suggest logical next steps based on conversation flow.

---

**Output Format**: Use clear markdown with code blocks, tables, and sections. Be specific with file paths, line numbers, and commit hashes where applicable.

**Important**: Analyze the entire conversation context to produce an accurate, detailed summary that would allow someone else to understand the complete session.
