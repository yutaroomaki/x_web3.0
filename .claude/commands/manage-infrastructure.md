Use the devops-engineer subagent to manage infrastructure and automation.

Infrastructure scope: $ARGUMENTS

The subagent should:
1. Define infrastructure requirements (compute, storage, network, database)
2. Create IaC templates (Terraform modules, CDK stacks)
3. Ensure modularity and reusability
4. Separate environments (dev, staging, production)
5. Apply security best practices (encryption, IAM, secrets management)
6. Configure monitoring and alerts (Prometheus, Grafana)
7. Generate infrastructure documentation
8. Create cost analysis reports

Output complete DevOps infrastructure suite with:
- Terraform modules (VPC, EKS, RDS, S3, ALB)
- Kubernetes manifests (Deployments, Services, Ingress with Kustomize)
- Ansible playbooks (server configuration, application deployment)
- Monitoring setup (Prometheus, Grafana dashboards)
- Secret management (Vault, AWS Secrets Manager integration)
- Deployment scripts (zero-downtime deployment, rollback)
- Infrastructure documentation (diagrams, runbooks)
- Cost optimization reports
